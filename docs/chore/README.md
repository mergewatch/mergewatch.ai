# docs/chore — chore & refactor plans

Phased plans for **chores / refactors / dependency bumps / infra / maintenance** — same format and lifecycle as [`docs/feat/`](../feat), authored by the [`/ship-feature`](../../.claude/skills/ship-feature.md) skill.

**Naming:** `YYYYMMDD-NN-<kebab-name>.plan.md` (the per-day sequence `NN` is shared across `docs/feat` and `docs/chore` so numbers don't collide on a given day).

Use this directory when the change isn't a new user-visible feature; use [`docs/feat/`](../feat) when it is.
